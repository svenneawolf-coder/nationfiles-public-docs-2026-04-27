________________________________________________________________________________

	本レポートはNationFiles Stability Index（NFSI）および7日間予測の手法・データプロビナンス・技術的追跡可能性を記述しています。大学・公的機関・認定監査人がシステムの信頼性と再現性を検証・確認するためのものです。NationFilesは完全な透明性を約束します：記載の全数式・パラメータは実装と一致し、虚構の値や事後調整はありません。NFSIは科学研究および監査の信頼できる参照として利用可能です。



________________________________________________________________________________
要約
	NFSIは3層の集約に基づく（レイヤー1：指標、レイヤー2：7日慣性を伴うテーマ別平滑化、レイヤー3：クラッシュモード含む加重最終スコア）。全重みと数式は固定され実装に反映。
	データは公的・ライセンスソース（例：世界銀行、ACLED、INTERPOL、GDELT）から取得。完全なプロビナンスとライセンスはデータインベントリおよび /legal/sources/ で利用可能。
	7日予測は90日履歴と複数時系列（世界指数、国別ニュースリスク等）を用いたVARベースの手法。手法は本ドキュメントおよび実装で一貫して文書化。
	ガバナンスにはバージョン管理、監査トレイル、外部監査人向けの定義済みプロセスを含む。



________________________________________________________________________________
範囲と対象読者
	対象：大学、研究機関、国内外の当局およびNFSIと予測の検証・認定を行う認定監査人。地理的範囲：全世界。NFSIデータベースの時間範囲：2026年2月以降、継続。範囲外：個別の投資・金融アドバイス。NFSIは安定性・リスクの評価に用い、金融商品の推奨には用いない。



________________________________________________________________________________
NFSIの整合性と信頼性
	NationFilesとNFSIは科学研究・監査・規制評価における信頼できる参照として設計されています。以下の方針は追跡可能性と防御可能性を強調します。
	虚構の値なし：すべてのNFSI値と予測は文書化された数式および本レポート・データインベントリに列挙されたソースのみから生成。手動上書きやブラックボックス補正はない。
	完全にコードベースかつ文書化：計算ステップ（レイヤー1–4）はすべてコードで実装され、定数・数式・疑似コードとともに本レポートで記述。
	プロビナンスと監査トレイル：データの出所、ライセンス、更新頻度はデータインベントリおよび法務ソースに文書化。モデル・データの変更は版管理され監査トレイルで追跡可能。
	ガバナンスにはバージョン管理、監査トレイル、外部監査人向けの定義済みプロセスを含む。
	アルゴリズム的解釈は公式評価を表さない。技術セキュリティ、監査サイクル（SCA/SAST等）、インフラはグローバル・セキュリティ＆レジリエンス声明（/legal/security/）に記載。生データの誤り訂正：data@nationfiles.com へ報告 → ソース確認 → DB修正 → 自動再計算。スコアの手動上書きは不可（独立性声明 /legal/independence/ 参照）。



________________________________________________________________________________
完全な透明性
	NationFilesはNFSIの手法・データ・計算における完全な透明性を約束します。隠れたパラメータ、不透明な補正、指数評価に関連する意図的な情報保留はありません。
	開示内容：
	本レポート内の全数式・計算ステップ（レイヤー1–4）、疑似コードおよび完全な定数表。
	NFSIに影響する全ソースの完全なデータインベントリ：出所、ライセンス、更新頻度、テーマ重み。法務ソース（/legal/sources/）へのアクセス。
	秘密の重み付けなし：重み行列（コネクタ重み、グループ）は文書化。要請に応じ認定監査人は監査契約に基づき技術成果物にアクセス可能。
	モデル・データの変更は版管理され、監査トレイル（タイムスタンプ、ハッシュ等）で追跡可能。国・日付ごとの計算ログは監査時に中間ステップ確認のため提供可能。
	ブラックボックスなし：公表されるすべてのNFSI値は文書化された数式・定数・入力データから再構成可能。下流の「秘密補正」や手動上書きはない。
	透明性はマーケティング主張ではなく文書化された実践です。本レポートとリンク参照がその基盤です。

________________________________________________________________________________
Layer 1 — Normalization and direction
	データソースごとに生値列（または算出深刻度）を使用。ソース全行で最小・最大を取得し、行ごとに指標の向きに応じて [0, 100] に正規化。



________________________________________________________________________________
Layer 2 — Today vs. previous day weight and aggregation rules
	コネクタ・国・日付ごとに関連する全 score_row を収集し集約、前日で平滑化（重み 0.6 / 0.4）。



________________________________________________________________________________
Layer 3 — Connector weights, groups, and effective weight
	各コネクタに設定重み（例40–100、データインベントリ参照）とテーマグループがあり、有効重みは現在スコアに依存。



________________________________________________________________________________
Layer 4 — Inertia weights and daily change cap
	レイヤー3の生スコア（nfsi_raw）は前日NFSIで平滑化；セキュリティ危機時は平滑化を行わない。

システムの流れ — 読みやすいNFSI記述
	以下は生データから公開NFSI値までの全フローを記述。
	生データはデータソース（コネクタ）ごとに取り込まれ、前処理（補完・タイムゾーン・重複除去）されます。各ソースは少なくとも1つの生値列または算出された深刻度を持つ行を提供します。
	レイヤー1（指標）：各データソースについて、全行の生値の最小・最大を算出します。スパンが0または未定義の場合は全行に中立の行スコア50を付与。それ以外は行ごとに生値を[0, 100]に正規化：正規化 = (生値 − 最小) / スパン × 100。指標により「生値が高い＝安定性が悪い」の場合は行スコア = 100 − 正規化、そうでなければ行スコア = 正規化。結果は[0, 100]にクランプし小数第2位に丸めます。これにより各行が統一された安定性スコアを持ちます。
	レイヤー2（コネクタ・国ごとの日次スコア）：各コネクタ・国・日付について、関連する全行スコアを収集。セキュリティグループ（グループ100）では日次スコアはこれらの値の最小（欠損は100でパッド）。その他のグループでは収集値の算術平均に固定ダミー0・100と欠損用パッド50を加えたものを採用。最終レイヤー2スコアは 0.6×日次スコア + 0.4×前日（前日：直近の利用可能値、なければ70または85）。結果は[0, 100]。欠損日はリカバリ（1日あたり0.2または1.0ステップ、キャップ95、最大90日）で補填可能。
	レイヤー3（国別NFSI）：各国・各日付について全コネクタのレイヤー2スコアが利用可能。欠損コネクタは中立スコア50（no_data）で含める。各コネクタの有効重みは グループ×(コネクタ重み/100)×更新乗数。ベーススコアは全スコアの加重平均（固定ダミー0・100を重み1で含む）。セキュリティスコアの最小が70未満の場合は紛争マルス（最大35）を減算。続いて脆弱性マルス、小国マルス、人口ボーナス、WGIプルを適用。生スコアは[1, 100]にクランプ。
	レイヤー4（日次平滑化）：生スコアを前日NFSIで平滑化。標準は前日80％・今日20％。no_dataが多い場合は今日45％または50％。前日からの変動は±3ポイントに制限。セキュリティ危機の場合は平滑化せず生スコアをそのままNFSI_Todayとして出力。結果がその国・その日付の公開NFSI値です。

日次指標とNFSIへの影響
	NFSIは準リアルタイムで安定性を反映する日次更新ソースを統合しています。ニュースイベント（GDELT）、メディアトーン、国別リスクシグナルを含み、レイヤー1で定義された指標を経て入り、レイヤー2・3を通じて日次値に影響します。
	関連する日次ソース：（1）GDELT — ゴールドスタイン尺度、平均トーン、国別イベントリスクレベル；（2）メディアセンチメント；（3）24h/48h窓の独自ニュースリスクシグナル。
	ゴールドスタイン・トーン・リスクレベルの平均からライブシグナルを形成。平均リスクが閾値を超えると控除を適用。結果は既存NFSI指数（例：ライブ40％、指数60％）で平滑化しノイズを抑制できます。
	簡略ロジック（疑似コード、実装と同一ではない）：
	LIVE_BASE = 50
	各ソースについて: LIVE_BASE += (Goldstein_avg × k1) + (Tone_avg × k2) + (Sentiment_avg × k2)
	IF risk_avg > threshold THEN LIVE_BASE −= (risk_avg − threshold) × k3
	LIVE_BASE = CLAMP(LIVE_BASE, 0, 100)
	IF 既存指数あり THEN SMOOTHED = 0.4 × LIVE_BASE + 0.6 × index

レイヤー1 — 指標の詳細
	レイヤー1ではデータソース・行ごとに0〜100の安定性スコアを算出。手法は指標タイプ（値ベース／件数ベース）に依存します。
	値ベース指標：ソースごとに生値列を使用。そのソースの全行で最小・最大を取得。行ごと：正規化 = (生値 − 最小) / (最大 − 最小) × 100。「生値が高い＝安定性が悪い」なら score_row = 100 − 正規化、そうでなければ score_row = 正規化。結果は [0, 100] にクランプ。
	件数ベース指標：行ごとに深刻度（例：人口当たり）を算出。必要に応じて参照人口（例：100万）を使用。スコアは一貫して0〜100にマッピング。
	レイヤー1数値例（値ベース、簡略）：
	ソース「紛争イベント」、国X：3時点の生値 5, 10, 20。最小=5、最大=20、スパン=15。行1：(10−5)/15×100 = 33.3 →「高い＝悪い」で score_row = 100 − 33.3 = 66.7。行2（値5）：score_row = 100；行3（値20）：score_row = 0。
	疑似コード（値ベース、高い＝悪い）：
	MIN_RAW = ソース全行のMIN(生値)
	MAX_RAW = ソース全行のMAX(生値)
	SPAN = MAX_RAW − MIN_RAW
	各行について:
	IF SPAN = 0 THEN score_row = 50
	ELSE 正規化 = (生値 − MIN_RAW) / SPAN × 100
	score_row = 100 − 正規化
	score_row = CLAMP(score_row, 0, 100)



________________________________________________________________________________
レイヤー2 — テーマ別集約と慣性
	レイヤー2はソース・国・日付ごとにレイヤー1スコアを日次スコアに集約し前日で平滑化します。
	(ソース、国、日付)ごとに関連する全 score_row を収集。セキュリティ関連ソース（グループ100）：日次スコア = MIN(スコア)。その他：日次スコア = スコア平均、安定重み付けのため固定ダミー0と100を含む。
	レイヤー2最終スコア：score_final = 0.6 × 日次スコア + 0.4 × 前日スコア。当日エントリがなければ回復値（最大95）を使い、欠損を無期限にペナルティしません。
	疑似コード：
	DAILY_SCORES = (ソース、国、日付)について score_row を収集
	IF グループ = セキュリティ THEN 日次スコア = MIN(DAILY_SCORES)
	ELSE 日次スコア = AVERAGE(DAILY_SCORES, ダミー0と100)
	prev = LAST_SCORE(国、ソース、前日)
	score_final = 0.6 × 日次スコア + 0.4 × prev
	score_final = CLAMP(score_final, 0, 100)

レイヤー3 — 最終スコア、危機モード、平滑化
	レイヤー3は全ソーススコアを国別NFSIに統合。各ソースにテーマグループと重みがあり、有効重みは現在スコアに依存。
	effective_weight = group_weight × (score_node / 100) × update_multiplier。グループは例：100（セキュリティ）、85、60、50、40；グループ−1は除外。
	NFSI_Base = SUM(score_node × effective_weight) / SUM(effective_weight)、重み1の固定ダミー0と100を含む。
	セキュリティスコア（グループ100）の最小値 < 70 の場合、紛争マルスを適用：malus = (70 − min_security) × 係数、上限（例35）。NFSI_Baseからこのmalusを減算。
	オプション：ガバナンスが強い（例 WGI est_total）と生スコアを上方に引き寄せ可能：final = raw + (100 − raw) × (est_total/100) × wgi_pull。
	日次平滑化（慣性）：セキュリティ最小 ≥ 25 なら NFSI_Today = prev × 0.8 + today × 0.2、日次変動最大 ±3。深刻なセキュリティ危機（最小 < 25）では平滑化を止めスコアが即時反応。
	疑似コード（コア）：
	各ソースについて: eff_w = group × (score_node/100)
	base = SUM(score × eff_w) / SUM(eff_w) ダミー0, 100含む
	min_sec = セキュリティグループのMIN(スコア)
	IF min_sec < 70 THEN malus = (70 − min_sec) × 1.0; base = base − malus
	raw = CLAMP(base, 0, 100)
	IF min_sec >= 25 THEN nfsi_today = 0.8×prev + 0.2×raw; 日次変動最大 ±3
	ELSE nfsi_today = raw
	図：NFSI三レイヤーの主要数式（疑似コードレベル）。

計算例（匿名化）
	重み行列の全文を開示せず説明：ある国でレイヤー2スコアが72、85、90の3指標がある日を想定。テーマグループは100、60、40で満値100。
	ステップ 1: 有効重み：72×1 = 72、85×0.6 = 51、90×0.4 = 36。重み合計 = 159。
	ステップ 2: 加重平均（ダミーなし）：(72×72 + 85×51 + 90×36) / 159 ≈ 80.2。ダミー0と100でも桁は同等。
	ステップ 3: Min(セキュリティ) = 72 ≥ 70 → 紛争マルスなし。NFSI_Base ≈ 80.2。
	ステップ 4: 前日 = 79。NFSI_Today = 0.5 × 80.2 + 0.5 × 79 = 79.6（簡略慣性）。
	実装では人口ボーナス、ガバナンスプル、慣性パラメータ等の追加要素を使用。本例は追跡可能性のためであり完全再現用ではない。

数式と手順の詳細

中心定数（参照）
	これらの値は式で使用：
	レイヤー1：スプレッドなし（min = max）または min/max なし → デフォルトスコア = 50。
	レイヤー2：当日重み = 0.6、前日重み = 0.4。ダミー値 0 と 100（各重み 1）。欠損エントリ（非セキュリティ）のパッド値 50。セキュリティグループ（グループ100）：欠損を 100 でパッド。1日あたり回復：セキュリティ 0.2 点、その他 1.0。上限 95。履歴なし開始：セキュリティ 70、その他 85。
	レイヤー3：紛争閾値 = 70、マルス係数 = 1.0、紛争マルス上限 35。参照人口 = 45M、pop-neg 乗数上限 2.0。小国閾値 = 5M。人口ボーナス：係数 0.5、上限 4.0。WGI pull = 0.95。No-data スコア = 50。脆弱性：WGI スケール 0–10、係数 3.0、上限 15。スコアフロア = 1。
	レイヤー4（慣性）：標準 80% 前日、20% 当日。欠損データ多い場合は 45% または 50%。日次変動最大 ±3。セキュリティ危機（min セキュリティ < 25）：慣性オフ、即時パススルー。



________________________________________________________________________________
レイヤー1 — データソースごとの行スコア
	入力：データソースごとに行のテーブル（国/日付/時間など）。ソースごとに生値列1つ（または算出深刻度）を定義。
	最小・最大：ソース全行で生値の min_raw と max_raw を算出。span = max_raw − min_raw。
	特殊ケース：span ≤ 0 または min/max 未定義の場合、全行で score_row = 50（中立）。
	行ごと正規化：normalized = (raw − min_raw) / span × 100。0 ≤ normalized ≤ 100。
	方向：「生値が高い＝安定性が悪い」なら score_row = 100 − normalized、そうでなければ score_row = normalized。
	クランプ：score_row = max(0, min(100, score_row))。小数第2位に丸める。score_row は行ごとに保存。



________________________________________________________________________________
レイヤー2 — ソース・国・日付ごとの日次スコア
	(ソース、国、日付)ごとの集約：この三重に対応する全レイヤー1スコア(score_row)を収集。国/日付でエントリが欠けている場合は前日とトレンドから推定。
	セキュリティグループ（グループ100）：日次スコア = MIN(収集スコア)。欠損値は 100 でパッド（「悪い事象1件で十分」）。
	その他グループ：ダミー 0 と 100 に欠損用パッド値 50 を加え、日付ごとの配列長を統一。日次スコア = この配列の算術平均。
	前日：(ソース、国、前日)の最後のレイヤー2スコアを yesterday として使用。欠損時は開始値 70（セキュリティ）または 85（その他）。
	平滑化：score_final = 0.6 × 日次スコア + 0.4 × yesterday。score_final = max(0, min(100, score_final))。小数第2位。
	回復：国にその日のエントリがない場合、回復を適用：セキュリティは score = min(95, score + 0.2)、その他は score = min(95, score + 1.0)。最大90日まで補充。



________________________________________________________________________________
レイヤー3 — 国別 NFSI（生スコア）
	寄与：(国、日付)ごとに全ソース（コネクタ）のレイヤー2スコアを提供。欠損コネクタエントリはスコア 50（no_data）と no_data フラグ。
	有効重み：各ソース n について eff_w_n = group_n × (scoreValue_n/100) × updateMult_n。group < 0 のソースは除外。
	加重平均：sum_weighted = Σ_n(score_n × eff_w_n) + 0×1 + 100×1（ダミー）。baseScore = sum_weighted / sum_weights。
	セキュリティ最小：minSec = MIN(score_n、group_n=100 かつ no_data でない n 全体)。セキュリティソースがなければ minSec = 80。
	紛争マルス：minSec < 70 なら conflictMalus = min(35, (70−minSec)×1.0)。rawScore = baseScore − conflictMalus。rawScore = max(0, min(100, rawScore))。
	人口乗数（負の効果）：popNegMultiplier = min(2, refLog/popLog)。脆弱性およびオプションで小国マルスに使用。
	脆弱性マルス：WGI_0_10 = est_total/10。governanceGap = 10 − WGI_0_10。fragilityMalus は [0, 15] にクランプ。rawScore = rawScore − fragilityMalus。
	小国マルス：pop < 5,000,000 なら smallPopMalus、[0, 25] にクランプ。rawScore = rawScore − smallPopMalus。
	人口ボーナス：popBonus = min(4, log₁₀(pop) × 0.5)。rawScore = rawScore + popBonus。[0, 100]。
	WGI pull：est_total がある場合(0–100)：finalScore = rawScore + (100−rawScore) × (est_total/100) × 0.95。nfsi_raw = max(1, min(100, finalScore))。小数第2位。



________________________________________________________________________________
レイヤー4 — 日次平滑化（慣性）
	前日：prevScore = (国、前日)の最後に保存された NFSI 値。
	セキュリティ危機：minSec < 25 の場合、平滑化なし：NFSI_Today = nfsi_raw（即時パススルー）。
	平滑化：prevScore が存在しセキュリティ危機でない場合：no_data 割合に応じて inertiaWeight = 0.45 / 0.50 / 0.80。smoothed = prevScore × inertiaWeight + nfsi_raw × (1 − inertiaWeight)。
	日次変動上限：diff = smoothed − prevScore。diff > 3 なら smoothed = prevScore + 3。diff < −3 なら smoothed = prevScore − 3。NFSI_Today = smoothed（小数第2位）。

	Formulas
	span = max_raw − min_raw (over all rows of the source)

レイヤー3 — 国別NFSI（生スコアと調整）
	式（実装順）
	Effective weight: effW_n = group_n × (scoreValue_n/100) × updateMult_n (scoreValue_n = connector weight/100).
	sum_weighted = Σ_n (score_n × effW_n) + 0×1 + 100×1; sum_weights = Σ_n effW_n + 2; baseScore = sum_weighted / sum_weights.
	minSec = MIN(score_n for group_n=100, not no_data); if none: minSec = 80.
	conflictMalus = min(35, max(0, (70 − minSec)×1.0)); rawScore = baseScore − conflictMalus; rawScore = clamp(rawScore; 0; 100).
	popNegMult = min(2, log₁₀(45e6)/log₁₀(max(pop, 100e3))). Fragility: WGI_0_10 = est_total/10; governanceGap = 10−WGI_0_10; popSens = 1/log₁₀(pop); fragilityMalus = min(15, governanceGap×popSens×3×popNegMult); rawScore −= fragilityMalus.
	Small country (pop < 5e6): smallPopMalus = min(25, (log₁₀(5e6)−log₁₀(pop))×4); rawScore −= smallPopMalus.
	popBonus = min(4, log₁₀(pop)×0.5); rawScore += popBonus; rawScore = clamp(rawScore; 0; 100).
	WGI pull: finalScore = rawScore + (100−rawScore)×(est_total/100)×0.95. nfsi_raw = max(1, min(100, finalScore)).
	Pseudo-code
	effW_n = group_n × (scoreValue_n/100) × updateMult_n
	sum_weighted = Σ(score_n × effW_n) + ダミー
	baseScore = sum_weighted / sum_weights
	IF minSec < 70 THEN conflictMalus; rawScore = baseScore − malus
	nfsi_raw = CLAMP(rawScore, 0, 100)

レイヤー4 — 日次平滑化（慣性）とクラッシュモード
	Formulas
	prevScore = last NFSI value (iso2, previous day). Security crisis = (minSec ≠ null AND minSec < 25).
	If security crisis: NFSI_Today = nfsi_raw (no smoothing).
	Else if prevScore exists: no_data_ratio = #no_data_connectors / #contributors. inertiaWeight = 0.45 if no_data_ratio ≥ 0.5; 0.50 if score without L1/L2 data; else 0.80. smoothed = prevScore×inertiaWeight + nfsi_raw×(1−inertiaWeight).
	diff = smoothed − prevScore; if diff > 3: smoothed = prevScore + 3; if diff < −3: smoothed = prevScore − 3. NFSI_Today = round(smoothed, 2).
	Pseudo-code
	prevScore = LAST_NFSI(iso2, 前日)
	SecurityCrisis = (minSec != null かつ minSec < 25)
	IF SecurityCrisis THEN NFSI_Today = nfsi_raw
	ELSE smoothed = prevScore × inertiaWeight + nfsi_raw × (1 − inertiaWeight); NFSI_Today = 変動上限(smoothed, ±3)

式と疑似コード
	年ごとに欠損を外挿で補完；コンセンサスは加重平均；生物学的成長制約を適用。
	外挿（2000年から目標年）
	入力: data = ソース[iso2] (年 → 人口)
	年でソート；minYear, maxYear
	minYearから目標年まで：欠損を外挿
	コンセンサス = 年別加重平均
	年別加重コンセンサス
	rawConsensus = ROUND( Σ(値 × 重み) / Σ(重み) )
	nullでない値を持つソースのみ合計に含む。
	生物学的制約（2024年以降）
	APIの急激な予測を和らげるため、年間成長率に上限（文脈に応じ例：最大2–3％）を設定。
	allowedGrowth = MAX(0.015, masterRate + 0.005)
	IF (rawConsensus - prevConsensus) / prevConsensus > allowedGrowth THEN 成長を制限

用語集 — 本レポートで使用する用語
	本用語集はレポートで使う主要用語を平易に説明。非専門家の理解を助けるため。
	NFSINationFiles Stability Index。国・日ごとの0–100正規化安定性指標。値が高いほど安定評価。複数データソースからレイヤー1–4で算出。レイヤー1第一処理層：各ソースの生値を行単位スコア0–100に正規化(Min-Max、方向「高い＝良い/悪い」固定)。レイヤー2第二層：行スコアをソース・国・日付ごとに日次スコアに集約し前日値で平滑化(例：当日60％、前日40％)。レイヤー3第三層：全ソーススコアから国別NFSI(生スコア)を算出。有効重みでの加重平均、紛争マルス、WGIプル、その他調整。レイヤー4第四層：日次平滑化(慣性)。最終NFSI日次値は前日と当日生スコアを組み合わせ(例：前日80％、当日20％)、日次変動に上限(例±3)。生値 (raw)0–100スコアに変換する前のデータソースの元の数値(例：紛争件数、割合、スケール値)。正規化生値を共通範囲(ここでは0–100)へ変換。Min-Maxでは(値−最小)/(最大−最小)×100。「高い＝悪い」時は反転可。スコア(行/日次/国)0–100に制限された値：行スコア(レイヤー1)、日次スコア(ソース・国・日付、レイヤー2)、国スコア＝NFSI(レイヤー3–4)。ゴールドスタイン尺度GDELTが使う−10(紛争的)〜+10(協調的)の尺度。NFSIロジックで安定性スコアに変換(ゴールドスタインが高い＝安定性が高い)。ダミー値集約時の固定加算値(0と100)。ソースが少なくても平均・重みが安定するようにする。慣性時間による平滑化：当日NFSIは前日に一部依存(例：前日80％、新規生スコア20％)し短期ノイズを抑制。リカバリ国・日付で新データがない場合、スコアを段階的に引き上げ(上限95)、欠損を無期限にペナルティしない。紛争マルスセキュリティ指標(紛争・暴力等)の最小が閾値(70)を下回るとNFSIから減算。セキュリティデータの役割を強調。WGI (Worldwide Governance Indicators)世銀の政府品質指標(法の支配、有効性等)。NFSIでは「WGIプル」でガバナンスが強いと生スコアを引き上げ可能。セキュリティグループ(グループ100)セキュリティ重視ソースのテーマグループ(紛争、渡航注意等)。日次集約では最小値を採用(1件の重大事象で十分)。コネクタ / データソースNFSIのレイヤー1–3に寄与する統合データソース(例：世銀、GDELT、ACLED)。各コネクタにテーマグループと重みあり。No-data特定日のソースの欠損値。中立スコア(例50)またはリカバリロジックで扱い、計算が失敗しないようにする。VAR(ベクトル自己回帰)7日予測の統計手法。複数時系列(NFSI、ニュースリスク等)の履歴で翌日以降を推定。データプロビナンスデータの出所と追跡可能性：どのソース、どのライセンス、更新頻度。本レポートのデータインベントリと法務ソースに記載。監査トレイル変更(データスナップショット、モデル版等)の追跡可能なログ。タイムスタンプとハッシュで監査・再現性に供する。再現性同じ入力と手法から同じ結果が得られる性質。文書化された式・コード・データプロビナンスで追求。
